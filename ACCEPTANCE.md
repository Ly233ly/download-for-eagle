## A. 核心自动化

- **A01**：IDM 完成受支持的视频下载后，系统自动创建导入任务。
- **A02**：Eagle 正在运行时，视频自动出现在当前资源库的“全部”中。
- **A03**：没有来源网页时仍然导入，Eagle 来源字段保持为空。
- **A04**：捕获到可靠来源时，Eagle 来源字段等于视频所在网页地址，不使用视频直链或猜测网址。
- **A05**：Eagle 未运行时任务保持“等待 Eagle”，启动后无需重新下载即可补导入。
- **A06**：导入完成后，下载目录中的原视频仍存在且内容未改变。

## B. 网站规则与来源

- **A07**：用户可在 Chrome 中开启或关闭当前网站自动导入。
- **A08**：开启网站后，扩展识别到的下载动作会保存网页来源。
- **A09**：关闭网站后，扩展识别到的下载动作会生成跳过指令；无法被浏览器识别的 IDM 动作仍按“无来源直接导入”处理。
- **A10**：可配置多个网站并正确处理常见子域名。
- **A11**：来源不确定时不得猜测；导入记录明确显示“未记录来源”。
- **A12**：已导入且有 Eagle 项目编号的记录可以事后补写来源，不重复导入文件。

## C. 文件与重复保护

- **A13**：非视频文件不会导入，并留下“非视频忽略”原因。
- **A14**：至少支持 MP4、MOV、MKV、WebM、AVI、M4V、MPEG、MPG、TS、M2TS 和 WMV。
- **A15**：相同内容的视频再次下载时跳过；不同内容即使同名也不能误判。
- **A16**：中文名称、空格、括号和较长 Windows 路径可以正确处理。
- **A17**：文件不存在、为空或仍在增长时不得提前导入。
- **A18**：同一路径在短时间内被 IDM 重复通知时只保留一个活动任务。

## D. 记录、恢复与占用

- **A19**：右下角存在托盘图标；左键可打开记录，右键显示 Windows 原生菜单，记录窗口可区分成功、等待、失败和跳过。
- **A20**：失败任务可以手动重试，错误原因对普通用户可理解。
- **A21**：程序或电脑意外关闭后，未完成队列不会丢失。
- **A22**：新下载通过轻量信号立即唤醒处理；空闲时不高频轮询，窗口隐藏后降低界面刷新频率。
- **A23**：默认记录保留 90 天、最多 10,000 条，并可手动清理。
- **A24**：助手和 IDM 接收器不会弹出终端窗口。

## E. 安全与回滚

- **A25**：默认流程的数据只保存在本机；媒体只从原网站下载到本机，除 Eagle 本机 API 和 GitHub Release 更新端点外不后台调用第三方处理服务；更新检查不得上传下载记录、来源或规则。
- **A26**：不保存 Cookie、账号密码、浏览器请求头或网页正文。
- **A27**：本机接口只监听 `127.0.0.1`，Chrome 扩展使用配对码、扩展来源和随机令牌认证。
- **A28**：安装前记录 IDM 原设置；停用后可以恢复。
- **A29**：卸载不会删除下载视频或 Eagle 项目。
- **A30**：网站规则、完整来源和本地路径不会写入对外诊断文件。

## F. 分享与一键安装

- **A31**：发行 ZIP 自带运行环境，未安装 Python 的 Windows 10/11 电脑也可运行。
- **A32**：一键安装器把完整程序复制到用户本地应用目录，创建桌面与开始菜单入口并注册卸载项。
- **A33**：IDM 对应设置为空或旧助手路径时自动配置为接收器与 `[File]`；发现其他安全软件配置时不覆盖。
- **A34**：安装前 IDM 原值只备份一次；卸载时仅在当前值仍指向本助手时恢复，避免覆盖用户后续修改。
- **A35**：安装器生成的自动配对凭据每台电脑不同、七天过期、仅允许 Chrome 扩展 Origin，并在成功配对后删除。
- **A36**：安装器自动打开 Chrome 扩展页面和正确文件夹；由于浏览器安全限制，最终“加载已解压”由用户亲自确认。
- **A37**：文件尚未稳定时界面明确说明会自动重试，单次等待不超过约 3 秒且最多 20 次；用户无需手动操作，也不会无限循环。
- **A38**：已经成功导入的同一任务不得因手动重试竞态变为“重复跳过”；历史误报记录在升级后自动恢复为“导入成功”。
- **A39**：普通临时错误最多自动重试 12 次；Eagle 长期未启动最多低频等待约 1 小时，达到上限后停止并允许用户手动重新开始。
- **A40**：同一网页产生多条重复视频请求时只作为一个来源候选；不得因候选自身重复而漏写可靠来源。
- **A41**：正常运行时任务管理器不得把本助手显示为 Python；主程序、后台、托盘和扩展应显示“下载中转站”及自有图标。
- **A42**：主界面和托盘菜单都提供检查更新入口；自动检查每天最多一次，未经用户确认不得安装。
- **A43**：更新清单必须通过内置 RSA 公钥验签；下载包的 SHA-256、实际大小或 GitHub 仓库地址任一不符时停止安装。
- **A44**：更新前保留旧程序目录，新版本必须通过本机版本健康检查；失败时恢复并重新启动旧版本。
- **A45**：更新不得删除或重置下载记录、网站规则、Chrome 配对和 Eagle 内容；0.6.0 后扩展可在后台版本变更后自行重载。

## G. 可见媒体候选与下载计划（1.0.0）

- **A46**：扩展默认展示当前标签页捕获到的媒体候选组，不需要先进入隐藏页面或打开调试开关。
- **A47**：每个候选组在下载前至少显示可识别标题、来源页面、封面或明确的预览失败原因、媒体类型、清晰度、音轨和已知大小。
- **A48**：用户选择普通直链媒体后，本机软件负责下载并持续记录进度、成功或失败原因；成功文件按选择进入现有 Eagle 队列，并保留可靠来源。
- **A49**：0.6.0 的 IDM 完成事件、无来源导入、网站开关、旧任务状态和扩展事件结构继续工作，旧客户端不会因新字段崩溃。
- **A50**：捕获同时支持 URL 扩展名、响应 MIME、Content-Disposition 和已知媒体请求特征；重复请求只形成一个可解释候选。
- **A51**：用户可以按媒体类型、扩展名和大小过滤；默认规则能排除常见图片、字体、脚本和小型噪声请求。
- **A52**：自定义正则无论有效、无效或灾难性输入都不能卡死扩展；错误规则会被禁用并显示原因。
- **A53**：站点规则、全局规则、临时忽略和手动强制捕获具有确定优先级，并能在界面解释最终结果。
- **A54**：候选预览支持站点封面、媒体首帧或安全的图标占位；不会仅以不可读长 URL 作为名称。
- **A55**：跨域、格式、编码、网络或 DRM 导致预览不可用时，界面显示具体类别，不把“无法预览”误报为“无法下载”。
- **A56**：候选可以按捕获时间、大小、类型、清晰度和名称筛选或排序，选择状态在列表刷新时保持稳定。
- **A57**：下载前能修改合法输出名，并看到将下载哪些媒体流、是否合并、是否导入 Eagle和最终容器；非法 Windows 文件名会被安全规范化。
- **A58**：原始 URL、请求类型和调试信息默认折叠但可展开复制；键盘操作、焦点和状态提示可用。

## H. 合并、HLS、DASH 与深度捕获（1.0.0）

- **A59**：本机 FFmpeg 可在不重新编码的场景使用 streamcopy 合并独立音视频，输出音画同步并通过 ffprobe 完整性检查。
- **A60**：可见主界面不加载浏览器 FFmpeg/WASM 或第三方在线处理页；普通直链、分轨和清单统一使用随软件安装的本机 FFmpeg。
- **A61**：HLS 主清单能展示全部变体、音轨、字幕和分辨率，并生成用户可检查的下载计划。
- **A62**：HLS 媒体清单正确处理普通分片、`EXT-X-BYTERANGE`、初始化段、相对地址、有限并发、重试和中止。
- **A63**：直播/持续增长清单能明确显示录制状态、已捕获时长和停止操作；停止后可以产出可播放文件。
- **A64**：AES-128 HLS 在获得页面本身可访问的密钥时能本地解密；SAMPLE-AES、Widevine 等 DRM 明确阻断且不尝试绕过。
- **A65**：HLS 合并产物经探测可播放后才进入现有 Eagle 导入流；失败时保留可诊断状态且不导入损坏文件。
- **A66**：DASH MPD 能解析 Period、AdaptationSet、Representation、BaseURL、SegmentTemplate/Timeline 和初始化段，并展示视频、音频及字幕选择。
- **A67**：在真实 B 站非 DRM 视频上，独立视频流和音频流属于同一候选组；用户一次选择即可由本机软件下载两路媒体、streamcopy、验证并把一个最终文件导入 Eagle。
- **A68**：深度搜索可以从 XHR、fetch、Worker、页面 JSON 和文本中发现常规请求监听遗漏的 HLS/DASH/媒体地址，并标出发现来源。
- **A69**：页面注入有明确启停、按标签页隔离且可撤销；禁用后恢复原 API 行为，不泄漏跨标签页数据，也不造成持续性能退化。

## I. 缓存、录制、路由与工具（1.0.0）

- **A70**：MSE/Cache 捕获能识别页面实际播放的缓存媒体，显示缓存范围和缺口，并在数据足够时导出可验证产物。
- **A71**：`MediaRecorder` 媒体录制和屏幕录制需要明确用户操作，显示录制来源、时长、大小和停止状态，并产出可播放文件。
- **A72**：WebRTC 流录制只处理用户选定且浏览器允许的流；权限拒绝、流结束和轨道变化均有明确结果。
- **A73**：所有可见主下载动作的最终路由固定为本机软件；失败进入同一任务错误/重试状态，不得静默回退 Chrome、第三方页面或造成重复下载。
- **A74**：IDM 路由由本机桌面助手执行，参数经过固定模板和路径校验；浏览器扩展不直接依赖或冒充 IDM 官方接口。
- **A75**：新版可见主界面不提供 Aria2、m3u8dl、自定义调用、send2local、MQTT 或完整浏览器下载器入口；上游兼容代码不得截获本机任务消息，凭据不会出现在日志或诊断导出中。
- **A76**：媒体播放控制、倍速、画中画、全屏和二维码功能只作用于用户当前选择的标签页或候选，并可恢复页面原状态。
- **A77**：M3U8/MPD/JSON/文本查看器、请求头与 User-Agent 覆盖、自定义命令都提供作用域、重置入口和错误提示。
- **A78**：Chrome、Edge 和目标 Firefox 版本的清单、权限、后台生命周期和核心下载路径均有自动或人工兼容证据。
- **A79**：所有新增设置都有默认值、版本迁移、导入导出和中英文文案；从 0.6.0 升级不会丢网站规则、配对或任务。

## J. 隐私、安全与 DRM（1.0.0）

- **A80**：Cookie、Authorization、Referer 等敏感下载上下文只在任务内存或受限临时记录中按需使用，默认不写入长期数据库和日志。
- **A81**：诊断导出会去除令牌、Cookie、完整本地路径和带查询参数的网址；用户可以预览将导出的内容。
- **A82**：外部目标、命令调用和第三方在线合并不出现在新版可见下载流程，也不会后台自行运行；本机 FFmpeg 参数按固定数组构造，不能形成命令注入。
- **A83**：扩展消息、回环媒体计划、停止/重试和本机命令均验证来源、配对令牌和允许状态转换；已删除临时组件完成/失败回传接口。
- **A84**：检测到 DRM 时清楚提示不支持；程序只删除明确位于任务专属临时目录的本程序中间文件，绝不删除用户原下载文件。

## K. 安装、升级、许可与回滚（1.0.0）

- **A85**：1.0.0 安装包包含目标浏览器扩展、受支持平台的 FFmpeg/ffprobe 及其许可证；安装后可离线完成普通音视频合并。
- **A86**：真实 0.6.0 数据目录升级后，队列、指纹、网站规则、配对、更新状态和旧 IDM 工作流保持可用。
- **A87**：健康检查覆盖后端、数据库迁移、扩展协议和 FFmpeg；任一关键项失败时恢复旧程序并保留数据。
- **A88**：卸载只删除安装器拥有的程序和临时目录，恢复属于本助手的 IDM 设置，并保留用户下载、Eagle 项目和可选数据备份。
- **A89**：若发行物复用或修改 cat-catch GPL-3.0 代码，则提供 GPL-3.0 许可证、版权说明、对应源码、构建/安装信息和获得源码方式。
- **A90**：FFmpeg、浏览器兼容代码及所有新增依赖都有版本、来源、许可证和发行包归属清单，自动检查不存在未登记二进制。

## L. 一次性 1.0.0 发布门禁

- **A91**：[功能对照矩阵](docs/FEATURE_PARITY_MATRIX.md) 的每一行都链接到实现任务、测试或人工证据，且没有未处理、无法解释的“缺失”。
- **A92**：0.6.0 全量回归、新增单元/集成/端到端测试、扩展语法与清单校验、C# 构建和安装包烟雾测试全部通过。
- **A93**：至少保留普通直链、HLS、DASH/B 站、直播停止、缓存捕获和一种录制模式的真实站点脱敏证据。
- **A94**：README、STATUS、TASKS、ACCEPTANCE、SECURITY、DEVELOPMENT、架构、决策、安装回滚、许可和版本号与最终行为一致。
- **A95**：维护者在完整功能、安全、许可、真实站点证据和回滚报告上一次性签字后才标记 1.0.0 并生成正式发行包。

每项验收至少保留自动测试、截图、任务记录或 Eagle 实际结果之一；没有验证的项目不得标记为完成。

## M. 1.1.x 方案 2 弹窗重构门禁

- **A96**：`popup.html` 只有一个应用根、一套运行时和“媒体 / 任务”导航；不得加载隐藏上游 popup DOM、旧 popup 事件层或第二套状态源。
- **A97**：同一明确内容 ID 的视频与音轨只在组内配对；不同 `groupKey`、清单身份或低置信度普通资源不得自动混组。
- **A98**：标题、封面、清晰度、音轨、文件名和预计动作在提交前可见；DRM、清单/直链混选与无效选择在提交前阻断。
- **A99**：多个方案从本机 `plans` 恢复并分别显示本机下载进度、阶段详情、错误和停止；一个调度器批量轮询，重开 popup 不丢任务。
- **A100**：深度搜索、缓存、三类录制、移动 UA、疑似密钥、当前页媒体控制、设置与独立窗口均有唯一可达入口；旧预览/二维码/手工解析器、自动下载快捷键/右键项、完整浏览器下载器和外部下载目标退出新版产品流程。
- **A101**：Chrome 实际工具栏 popup 与选定方案 2 在同一 660×574 视口对照，P0/P1/P2 视觉和交互问题全部修复。

## Q. 1.2.3 通用内容页解析与字节分片重建
