# 当前进展

更新时间：2026-07-17

## 当前版本

0.6.0。软件统一命名为“下载中转站”，使用独立冻结后台和自有图标，任务管理器不再把本助手显示为 Python；源码已按 MIT 许可证公开，并加入安全的一键更新。

## 已完成并验证

- IDM 下载完成接收、SQLite 持久队列、Eagle 导入、离线重试、内容去重与无来源直接导入。
- Chrome 来源捕获、网站规则、离线事件补发、手动记录、单次忽略和持久配对。
- 可视化网站规则列表；右下角菜单可直接打开规则。
- 助手未运行时由 IDM 接收器静默启动；来源最多短暂等待 4 秒，不阻塞无来源导入。
- Windows Forms 原生托盘宿主，避免旧 Python 原生回调导致的 `0xc0000409` 崩溃。
- PyInstaller 独立后端包含 Python/Tcl/Tk；发行包不依赖接收者电脑的 Python。
- 一键安装器自动复制程序、配置 IDM、创建桌面/开始菜单入口、注册卸载项并启动助手。
- 安装器保护 IDM 既有杀毒软件配置，并保存原值供卸载恢复。
- 每台电脑生成一次性 Chrome 自动配对凭据；成功使用后立即删除。
- 文件尚未稳定时明确显示“等待自动重试”，约 2–3 秒内自动继续；已导入任务不能再被手动重试改成“重复跳过”。
- 升级数据库会把此前“同一任务已成功但误标重复”的记录自动恢复为“导入成功”。
- 自动重试设有硬上限：文件稳定检查最多 20 次，其他临时错误最多 12 次，Eagle 等待最多约 1 小时。
- 同一网页的重复来源事件会先合并，再与其他网页比较；不同网页确实无法区分时仍保持来源为空。
- 主程序、冻结后台、托盘、窗口、安装器和 Chrome 扩展统一使用“下载中转站”名称与图标。
- 日常运行使用 `下载中转站后台.exe`，不再启动开发机 `pythonw.exe`；任务管理器文件说明与产品名均为“下载中转站”。
- 主界面与托盘菜单提供检查更新；每天最多静默检查一次，用户确认后才安装。
- 更新清单使用离线私钥签名，客户端内置公钥验签，并核对 ZIP 的 SHA-256、大小和 GitHub 仓库地址。
- 更新前暂存旧程序目录，新版本需通过 20 秒本机健康检查；隔离测试已确认故障时恢复旧版本和配对信息。
- Chrome 扩展每 30 分钟低频确认一次后台版本，只在后台版本更新时自行重载。
- 源码已发布到 `Ly233ly/download-for-eagle`，采用 MIT License；运行数据、配对信息、私钥和发行包不进入 Git。
- 49 项自动测试、扩展语法、C# 编译、隔离安装、独立后端、IDM 接收器、自动配对、更新验签和回退均通过。
- 0.6.0 已在当前电脑正式安装；4 条历史、2 条规则和 Chrome 配对迁移后保持不变，IDM 已指向正式接收器，后台健康检查为 0.6.0。
- 真实 IDM 下载、真实 Eagle 导入和 Behance 来源写入此前已验证成功。

## 仍需接收者手动完成

- Chrome 出于安全限制要求用户亲自开启开发者模式并选择“加载已解压的扩展程序”。安装器会自动打开页面和正确文件夹，加载后无需再输入配对码。
- 若 IDM 的“病毒扫描程序”已被安全软件占用，安装器不会覆盖，需要用户决定是否改用该入口。

## 已知边界

- IDM 不提供来源网页；浏览器未识别到的下载会按无来源直接导入。
- 未发布到 Chrome 应用商店，Chrome 会显示正常的开发者模式扩展提示。
- 0.5.0 及更早版本需手动安装一次 0.6.0，并手动重载扩展一次；之后才能使用完整的一键更新与扩展自行重载。
