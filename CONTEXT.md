# 下载中转站领域上下文

## 产品目标

下载中转站把浏览器中发现的媒体变成用户可识别、可选择、可可靠下载和合并的本机文件，随后通过 Eagle 官方本地 API 导入当前资源库。1.0.0 曾完成 `cat-catch` 行为研究与迁移；1.2.0 起产品边界收敛为“扩展发现与提交、本机软件执行与交付”，不再把上游工具箱本身作为运行时目标。

## 领域词汇

- **媒体候选**：一次网络请求或页面分析发现的可下载资源描述，不等同于已下载文件。
- **媒体候选组**：属于同一内容的一个或多个媒体候选，例如 B 站同一视频的多档视频流、音频流和封面。
- **播放器视觉身份**：由 frame、播放器序号、去签名参数后的媒体路径和时长桶组成的短时稳定键；用于把同一播放器的轮换请求和预览帧归到同一候选组，不作为下载 URL。
- **瞬时视频帧**：页面 canvas 或播放器可见区域裁剪得到的当前视频画面；只保存在扩展 service worker / popup 内存，不写媒体缓存、数据库或诊断。
- **捕获会话**：同一标签页、主文档导航周期和页面上下文中的候选集合。
- **媒体流**：候选组中的具体视频、音频、字幕或清单流，带编码、分辨率、码率和预计大小等属性。
- **结构化播放器目录**：页面播放器已经解析出的格式集合，例如 YouTube `streamingData`；适用于 MSE 导致 DOM/网络层只暴露占位地址或探测分片的站点，必须按明确内容 ID 和流 ID 归组，不能猜测签名地址。
- **桌面解析候选**：浏览器能证明内容身份但没有可复用媒体 URL 时形成的单一候选。`resolver=youtube` 来自明确的播放器画质目录；`resolver=page` 来自 blob 播放器附近的同源稳定内容链接。扩展只展示并提交，桌面固定解析器把页面/所选高度转换为瞬时音视频 URL，随后进入统一 FFmpeg 状态机。
- **下载方案**：用户确认的输出计划，包含所选媒体流、固定的本机下载方式、合并方式、目标文件名以及是否导入 Eagle。
- **任务临时文件**：本机软件在 `下载中转站/临时/<planId>` 内创建的中间输出；不包含浏览器或用户原下载文件。
- **最终媒体文件**：由本机软件下载、解密（仅非 DRM）、合并和验证完成的文件；可以进入 Eagle 队列，也可以按“仅下载”直接交付。
- **可见下载**：下载前默认展示标题、封面或预览、类型、清晰度、音轨、大小和输出文件名，不要求用户从 URL 猜内容。
- **本机下载与合并**：使用安装包内的 FFmpeg/ffprobe 访问原网站媒体地址、streamcopy、验证并生成最终文件；这是所有主下载动作的唯一路径。
- **下载线路**：单个远程媒体任务使用的直连或 HTTP 代理选择。自动模式从 Windows 当前用户系统代理生成，手动模式来自用户明确配置；它不作用于 Eagle、本机 API 或控制信号。
- **来源网页**：媒体所在页面，用于 Eagle 的来源字段；它不是媒体直链。
- **DRM 阻断**：识别到 Widevine、PlayReady、FairPlay、SAMPLE-AES 等受保护内容后停止下载/解密，并向用户解释原因。
- **播放分片**：播放器加载过程中的固定字节区间、`.m4s` 或 `.ts` 传输单元；默认不进入可下载内容列表，只能在筛选器中显式查看紧凑诊断项。
- **重建直链后备**：从成对字节区间参数中只移除传输偏移、保留全部签名参数得到的完整媒体地址；只有结构和响应证明可重建时存在。若同页已有稳定桌面解析候选，它退出默认信息主线但可在技术分片中查看。
- **补导 Eagle**：`completed_local` 最终媒体不重新下载，直接创建关联 Eagle job 并进入既有导入状态机。

## 不使用的近义词

- 不把媒体直链称为“来源”；来源专指页面地址。
- 不把视频流和音频流称为两个独立“视频”；它们是同一媒体候选组中的媒体流。
- 不把扩展称为下载器；扩展是媒体发现与任务控制界面，本机软件才执行下载。
- 不把 `.m4s`/`.ts` 传输分片称为多个视频；它们必须收敛到清单或一个明确的“仅分片、不可直接下载”候选。

## 固定边界

- 用户原文件不移动、不删除、不修改。
- 不直接修改 Eagle `.library`。
- IDM hook 只入队和唤醒。
- DRM 只识别和阻断。
- 未经明确确认不持久化 Cookie、Authorization 或完整请求头；日志和诊断永不包含这些信息。
- 完整签名媒体 URL 和敏感请求上下文只存在于当前本机任务内存；软件重启后不得从脱敏地址猜测恢复。
- 代理账号密码不持久化；手动代理只接受无认证 HTTP/Mixed 主机端口，诊断只记录模式、来源和脱敏端点。
- 扩展运行时不包含浏览器下载器、在线 FFmpeg、录制器、移动 UA、密钥面板、旧解析/预览页或第三方下载路由；这些历史源码只保留在许可归档中。
- 当前发行物统一采用 GPL-3.0；历史 MIT 和固定上游通知只作为不可删除的版权与来源记录，不构成第二套产品许可。
