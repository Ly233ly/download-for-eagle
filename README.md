# 下载中转站

<p><img src="assets/download-transfer-station.png" width="128" alt="下载中转站图标"></p>

把 IDM 下载完成的视频自动中转到当前 Eagle 资源库的“全部”。捕获到视频所在网页时会保存为 Eagle 来源；没有来源也会直接导入。

## 分享给别人

直接发送项目 `release` 目录里的发行 ZIP。接收者不需要安装 Python：

1. 解压整个 ZIP。
2. 双击 `一键安装.exe`，点击“一键安装”。
3. 安装器自动配置 IDM、创建快捷方式、启动助手，并打开 Chrome 扩展管理页和扩展文件夹。
4. 在 Chrome 开启“开发者模式” > “加载已解压的扩展程序” > 选择已打开的 `chrome-extension` 文件夹。

扩展加载后会使用每台电脑独有的一次性凭据自动配对，不需要输入六位码。由于 Chrome 的安全限制，“加载已解压的扩展程序”必须由用户亲自确认，不能由安装器代点。

安装器为兼容旧版本，程序目录仍使用 `%LOCALAPPDATA%\IDM-Eagle自动导入助手`；对外名称、快捷方式、任务管理器和托盘均显示“下载中转站”。它会备份 IDM 原设置；若发现 IDM 已配置其他杀毒软件，不会强行覆盖，并会给出手动提示。卸载入口位于 `Windows 设置 > 应用 > 已安装的应用` 和开始菜单。

## 日常使用

1. 正常打开 Eagle。
2. 在 Chrome 中使用 IDM 下载视频。
3. 下载结束后自动导入；助手没开时，IDM 接收器会静默启动它。

右下角助手图标：左键打开记录，右键可打开记录、网站规则、立即检查或退出。主界面的网站规则列表可新增、启停、控制子域名或删除规则。未列出的网站默认关闭自动来源捕获。

扩展还提供“为下一次下载记录当前网页”和“本次下载不导入”。浏览器来源暂时发送失败时会在本地排队并快速补发；同一网页的重复视频请求会先合并，避免可靠来源被误判为不确定。助手会短暂等待来源，超时后仍然无来源导入。文件未稳定时每 2–3 秒自动重试，最多 20 次；其他临时错误最多 12 次，避免无限循环。

## 保护与边界

- 原下载文件始终保留，不移动、不删除、不修改。
- 重复文件按完整内容哈希判断。
- Eagle 关闭时任务持久保存，重新打开后自动继续。
- 不保存 Cookie、账号、请求头或网页正文，不向外部服务上传数据。
- 不修改 Eagle `.library` 内部文件，只调用 Eagle 官方本地 API。
- IDM 不提供网页地址；若浏览器完全没有识别到某次下载，该视频会按“无来源直接导入”处理。

## 环境与源码运行

发行包支持 Windows 10/11，需要已安装 IDM、Eagle 和 Chrome，不要求 Python。

源码开发模式需要 Python 3.11+，运行时代码仅使用标准库。开发入口、打包和测试见 [DEVELOPMENT.md](DEVELOPMENT.md)，详细验收见 [ACCEPTANCE.md](ACCEPTANCE.md)，安装与回滚见 [docs/INSTALLATION_AND_ROLLBACK.md](docs/INSTALLATION_AND_ROLLBACK.md)。
