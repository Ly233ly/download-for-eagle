# 当前扩展 popup 审计

- 日期：2026-07-18
- 表面：Chrome 工具栏扩展 popup
- 任务：发现当前页面媒体，选择正确视频/音轨，下载、合并并导入 Eagle
- 证据：[`01-current-popup.png`](01-current-popup.png)，662 × 574
- 总体健康：不健康；核心能力存在，但信息架构、状态和操作入口相互冲突。

## 步骤 1：打开 popup 并理解当前状态

健康度：差。

### 可确认的优点

- 产品名、版本、配对状态和媒体数量在首屏可见。
- 当前网站规则明确说明“不影响下载”，避免把来源记录与下载能力混为一谈。
- 深度搜索、缓存捕捉、下载、筛选等已有能力没有被删除。

### UX 问题

1. `下载中转站` 主区域和底部 cat-catch 区域是两套完整导航，用户无法判断哪套是主界面。
2. 页面说“0 项媒体”，内容卡又像一个可下载对象；“未发现媒体”和“已选最佳视频与音轨”出现在同一视觉语境，状态自相矛盾。
3. 配对输入占据首屏最强位置，却没有解释配对对象、为什么需要或配对后能获得什么。
4. `保存来源并自动导入 Eagle`、`记录当前页`、`忽略下一次导入`与下载选择处于同一层级，主任务被配置项打断。
5. 内容卡内部同时出现文件名输入、两个合并方式、停止、刷新和进度，用户被迫先理解实现方式。
6. 禁用按钮与普通按钮主要靠灰度区分，缺少原因；例如不能判断是“没配对”“没媒体”还是“不支持”。
7. 底部两层固定栏塞入 15 个以上入口，按钮尺寸和间距不足，且遮挡/压缩主内容。
8. `当前页面 / 其他页面 / 其他功能`与上方内容卡没有明确的数据关系；底部“数据载入中”也与上方“0 项媒体”冲突。
9. 文案混用“本机下载/合并”“浏览器在线合并”“在线合并”，结果和实现方式没有统一表达。
10. 主页面没有内容级归组、清晰度、音轨和最终输出摘要，用户仍然不知道将下载什么。

### 可见的视觉问题

- 662px 宽弹窗仍显拥挤，根因不是宽度不足，而是重复导航和过量常驻操作。
- 深蓝底色上同时使用多个边框容器、胶囊、实心按钮和固定栏，视觉层级过多。
- 主标题、版本、媒体计数、网站名、产品卡标题、说明文字都在争夺首屏注意力。
- 白色进度条过亮且没有进度文本，容易被误认为分隔线。
- 底部控件使用另一套字体、圆角、蓝色和密度，明显来自不同产品。

### 无障碍风险

- 多个小按钮低于舒适点击尺寸，尤其底部工具栏。
- 禁用/启用和当前状态高度依赖颜色与灰度。
- 两套导航可能造成重复、混乱的键盘焦点顺序和屏幕阅读顺序。
- 动态的“数据载入中”、媒体计数与全局状态是否通过 `aria-live` 通知无法从截图确认。
- 深色主题的次级灰字、禁用文字和边框需要用实际颜色值验证对比度。
- 整页重绘导致焦点、输入草稿和滚动位置丢失的风险需要通过键盘实测确认。

## 最高优先级机会

1. 删除旧 UI 的重复可见层，只保留一套 `媒体 / 任务 / 工具` 导航。
2. 把首屏收敛为一个用户结果：看见内容、确认推荐版本、下载并导入。
3. 网站规则和高级下载目标进入设置/更多菜单；空状态再就地暴露发现工具。
4. 下载按钮前显示“1080P 视频 + 中文音轨 → 本机无损合并 → Eagle”等可读摘要。
5. 每个任务独立显示阶段、进度、错误和停止，替代全局单条进度。

## 证据边界

截图只覆盖空媒体/未配对状态。候选列表、版本选择、下载中、合并、失败、完成、键盘操作、缩放和窄宽度仍需在实现后通过 Chrome 实机状态逐项验证；本审计不声明完整 WCAG 合规。
