# 安装、更新、故障排查与卸载

## 推荐安装（分享版）

1. 把发行 ZIP 完整解压。
2. 双击 `一键安装.exe`，点击“一键安装”。
3. 安装器自动完成：
   - 为兼容旧版本，复制到内部目录 `%LOCALAPPDATA%\IDM-Eagle自动导入助手`；对外统一显示“下载中转站”；
   - 生成每台电脑独有的一次性 Chrome 配对凭据；
   - 备份并配置 IDM 的 `VScannerProgram` 与 `VScannerParameters=[File]`；
   - 创建桌面、开始菜单和 Windows 已安装应用条目；
   - 启动助手，打开 Chrome 扩展管理页和正确扩展目录。
4. Chrome 中开启“开发者模式”，点击“加载已解压的扩展程序”，选择已打开的 `chrome-extension` 文件夹。
5. 扩展首次连接时自动配对并销毁一次性凭据。

Chrome 的最后一次加载确认属于浏览器安全边界，安装器不能代替用户点击。安装器会把扩展目录复制到剪贴板，便于在文件选择窗口粘贴。

## IDM 保护规则

- IDM 对应设置为空：自动配置。
- 已指向旧版 `IdmEagleHook.exe`：升级到新安装位置，并把最初状态按空值保存。
- 已指向其他程序：视为安全软件配置，不覆盖；完成页提示手动处理。
- 原值只备份一次，避免更新时把本助手路径误当成用户原设置。

## 更新与修复

重新运行新版 `一键安装.exe` 即可覆盖程序文件并保留 `%LOCALAPPDATA%\IdmEagleAutoImport` 中的队列、网站规则和配对状态。更新扩展文件后，到 `Chrome > 扩展程序 > 管理扩展程序` 点击一次“重新加载”。

## 正常使用

- Eagle 开着：IDM 完成视频下载后通常几秒内导入。
- 助手未开：接收器自动静默启动，不要求开机启动项。
- Eagle 没开：记录保持“等待 Eagle”，启动 Eagle 后自动继续；低频等待最长约 1 小时，之后停止并允许手动重试。
- 浏览器来源稍晚：助手最多短暂等待 4 秒；仍无来源则直接导入。
- 来源发送失败：扩展先保存在本地，1 秒、2.5 秒、5 秒快速补发，并保留 30 秒兜底。
- 文件尚未稳定：每 2–3 秒自动重试，最多 20 次；其他临时错误最多 12 次，均不会无限循环。

## 常见问题

### Chrome 没有自动配对

确认下载中转站是 0.5.0、扩展是 0.5.0，并且安装后七天内首次加载。若凭据已过期或被消费，可使用助手显示的六位配对码手动配对。

### 下载后没有记录

进入 `IDM > 选项 > 下载 > 病毒检查选项 > 编辑`，核对程序为安装目录中的 `IdmEagleHook.exe`，参数必须是 `[File]`。若这里属于杀毒软件，请不要覆盖，需选择其他接入方案。

### 一直等待 Eagle

打开 Eagle 并进入目标资源库，再在助手点击“立即重试（可选）”或在托盘菜单点击“立即检查”。

## 卸载与回滚

从 `Windows 设置 > 应用 > 已安装的应用` 或开始菜单运行“卸载助手”。卸载程序会：

- 先退出助手；
- 仅当 IDM 当前仍指向本助手时恢复安装前值；
- 删除桌面/开始菜单入口和程序目录；
- 保留 `%LOCALAPPDATA%\IdmEagleAutoImport` 中的历史、规则和指纹，方便以后重装。

卸载不会删除下载视频、Eagle 项目或 Chrome 官方 IDM 扩展。Chrome 中加载的个人扩展需在扩展管理页手动移除。
