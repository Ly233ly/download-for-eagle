# 33 仅下载动作真实性

状态：已实现，等待 1.1.5 整体发行复验。

## 问题

popup 处于“需要配对”时，“下载并导入 Eagle”会禁用，但“仅下载”仍可点击。`downloadOnlyForGroup` 在校验失败后直接返回，外层 `downloadOnly` 却无条件显示“下载已开始”，因此本机没有收到 `/api/media/plan`、SQLite 没有新计划、两个任务列表都为空。

当前开发机还同时存在正式安装副本和仓库开发副本。Chrome 实际加载仓库副本；该副本按安全规则不含 bootstrap，一次性自动配对只对正式安装副本生效。

## 实现

- 新增 `startValidatedTask` 无 DOM 边界，校验失败时不调用创建器并返回 `started=false`。
- 单项成功提示依赖真实计划对象；没有计划就不显示成功、不切换任务页。
- 单项和批量“仅下载”使用与主按钮相同的校验结果决定是否禁用。
- service worker 暴露 `autoPair`，popup 每次连接检查都主动尝试一次自动恢复。
- 仓库 bootstrap 继续为空；正式安装器的单机凭据边界不放宽。

## 验收

- Node 回归证明未配对校验不会调用创建器；有效校验只创建一次并返回计划。
- 静态扩展回归证明存在 `autoPair` 路由、连接检查会调用它、成功 toast 受计划对象条件控制。
- Chrome 重载 1.1.5 后完成配对；点击“仅下载”必须同时在 popup 和桌面出现同一真实任务，最终为 `completed_local/100`，并能打开保存目录。
