# 41 媒体身份一致性

状态：实现与自动回归通过；1.1.9 实机证明错配防线有效，但安全留空造成的预览覆盖问题由任务 43 / 1.1.10 接替。

## 现场

- popup 选中的抖音卡片显示当前播放器画面与 `45:13`。
- 桌面计划记录同一候选 `duration=2713.034`，但最终文件 FFprobe 实测 `708.141475`，本机首帧也是另一段视频。
- FFmpeg 按计划中的 URL 正常下载；错误发生在计划创建前的浏览器候选身份关联。

## 根因

内容脚本无法用网络请求 URL 匹配任何 `<video>` 来源时，会回退到当前播放或可见播放器。抖音页面同时保留多个预加载/MSE 播放器，后台 11:48 视频的真实网络 URL因此继承了可见 45:13 视频的画面、时长和稳定组键。

## 实现

- `resolveVisualMatch` 只接受规范化后同主机、同路径的播放器来源；签名查询参数可以轮换。
- 未匹配请求返回 `visualMatch=none`，不绑定帧、缩略图、时长、尺寸或内容组。
- 结构化播放器目录仍可使用其明确提供的内容 ID、封面、时长和清单。
- 本机对非清单直链执行第二层身份检查：FFprobe 输出时长与声明时长相差超过 `max(3 秒, 8%)` 时，删除程序临时输出并返回 `output_duration_mismatch`，不进入完成目录或 Eagle。

## 回归

- JS 回归模拟一个可见 2713 秒播放器、一个隐藏 708 秒 blob 播放器和后者的未匹配网络 URL；修复前因缺少可信匹配边界失败，修复后未匹配请求不再继承可见播放器。
- Python 集成回归用 1 秒真实视频冒充 12 秒候选；修复前错误文件达到 `completed_local`，修复后进入 `retry/output_duration_mismatch` 且完成目录无文件。
- 91 项全量 unittest、扩展 JavaScript 语法和 Chrome/Firefox 双清单通过。
