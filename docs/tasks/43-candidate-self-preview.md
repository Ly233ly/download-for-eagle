# 43 无静态帧候选预览自身直链

状态：实现与自动回归通过，等待 1.1.10 Chrome 实机闭环。

## 现场

- 抖音 popup 捕获 4 个普通视频候选，只有 1 个显示真实帧，其余显示默认下载图标。
- 当前页面 DOM 实测只有 1 个可见 `<video>`；网络层候选数量大于播放器数量，1.1.9 的 URL 精确匹配只能安全绑定当前播放器对应候选。
- 直接把当前播放器帧复制给另外三个候选会重新引入“选 A 下 B”，因此缺图不是站点白名单，而是安全留空缺少第二展示路径。

## 实现

- 视图纯逻辑 `previewMediaUrl(group, selection)` 从当前选择中取视频候选，只接受 `http/https`。
- 有可信 frame/poster/结构化 thumbnail 时继续优先显示静态预览；没有时创建静音、内联、metadata 预加载的视频元素，解码该候选自己的开头画面。
- 版本选择与预览共用同一 `selection`；选择变化后预览 URL 与桌面计划提交 URL 同步变化。
- 远程预览加载失败回退默认占位；音频、blob 与非法 URL 不进入视频回退。下载路线仍固定为本机软件。

## 回归

- Node 回归先因缺少 `previewMediaUrl` 失败；实现后证明第二个候选被选择时只返回第二个 URL，切回第一个时同步更新，音频返回空。
- Python 扩展回归验证 UI 包含 metadata 预加载、元数据后定位代表帧、错误回退和对应 CSS。
- 91 项全量 unittest、全部扩展 JavaScript 语法和 Chrome/Firefox 双清单通过。
