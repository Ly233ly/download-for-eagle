# 73 通用信息流候选绑定

状态：实现完成，发行与实机复验由任务 76 跟踪。

## 问题

信息流页面只渲染少量真实视频卡片，但 blob/MSE 会产生大量轮换 HLS/DASH/音视频请求。网络 URL 无法与 `video.currentSrc=blob:` 精确匹配时，旧归组把每条完整清单当作一个独立内容，并回退标签页标题，用户无法判断下载对象。

2026-07-20 现场样本中，页面 DOM 有 4 个 `<video>`（2 个可视），popup 却显示 18 个默认媒体组并隐藏 42 个分片。每个播放器实际都能从最近内容容器取得独立永久链接、正文、poster 和时长。样本站点只用于复现，不能成为实现条件。

## 实现

1. `selectContentTitle` 从容器的语义说明、标题、可读正文与图片说明中选取内容标题，并过滤计数、时间轴、相对时间和控件噪声。
2. blob/MSE 播放器继续由 `chooseContentPageUrl` 证明同源永久链接，形成带自身标题、封面、时长的 `resolver=page` 候选。
3. `partitionGroups` 在同一标签页/来源域已有内容绑定候选时，把无稳定身份的孤立 HLS/DASH/音视频归为 `technicalOnly`；默认隐藏，诊断视图紧凑显示且 `validateSelection` 阻止提交。
4. 没有内容绑定候选时不执行上述降级，唯一完整清单保持可下载。
5. 工具栏角标与 popup 都使用 `groupCandidates → partitionGroups` 的默认可见数量。

## 自动验收

- 一个信息流含 3 个内容绑定候选和 12 个未关联 HLS 时，默认只显示 3 个；诊断视图保留 12 个技术项且下载校验返回 `unbound_playback`。
- 同一个未关联 HLS 在没有页面候选时继续可见。
- 内容正文样本必须胜过标签页标题；纯计数和时间轴不能成为标题。
- A173–A176 为正式验收标准。
