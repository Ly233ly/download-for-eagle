# 45 强媒体别名归并

状态：实现与自动回归通过，等待 1.1.11 Chrome 实机闭环。

## 问题

同一视频可由主 CDN 和备用 CDN 暴露为不同签名 URL。能与页面播放器来源精确匹配的主请求带真实帧、时长和稳定组键；备用请求没有这些元数据，旧归组把它当成第二个内容，造成同大小的两个列表项且只有一个有图。

## 实现

- 候选规范化保留会话内 `contentIdentity`；后台优先从 Digest、Content-MD5 或非弱 ETag 生成该身份。
- 没有响应身份时，只对同页同类型、长媒体路径完全相同且总字节数完全一致的地址建立强别名。
- 强别名唯一指向一个显式播放器组时，无组键备用请求并入该组；若多个显式组争用同一别名则不归并。
- 组内 CDN 别名折叠成一个版本，按真实预览、播放器组键、时长、质量元数据和捕获时间选择最丰富记录。
- 不按域名写白名单；文件大小相同但路径/响应身份不同的媒体保持独立。

## 验收

- 同路径、同大小、不同 CDN 主机的一有图/一无图候选形成 1 个内容组和 1 个下载版本。
- 响应身份相同但 URL 路径不同的 CDN 地址也形成 1 个版本。
- 仅大小相同但路径不同的媒体保持 2 个内容组。
- 归并后标题和预览来自信息更完整的播放器候选；下载仍提交该候选真实原始 URL 给本机软件。

对应验收：A141。
