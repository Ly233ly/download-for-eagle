# 65 抖音当前主播放器专用适配

状态：ready-for-human

## 根因

现场精选页同时存在两个已就绪播放器：视口内当前视频为 223.237 秒、1920×1080、HTTP `currentSrc`；下一条预加载视频位于视口下方，时长 49.041 秒且为 `blob:`。通用页面解析旧逻辑跳过所有 HTTP 视频，只处理 blob，因此把下一条内容的 0:49、画面和导航页交给下载任务。

yt-dlp 的 `DouyinIE` 只接受 `/video/<id>`，不接受 `/jingxuan` 或 `/jingxuan?modal_id=<id>`。错误中的 `[媒体地址]` 是安全脱敏结果，不是实际提交的占位符。

## 实现

- `selectPrimaryPageVideo` 按视口交集、播放状态、播放进度和面积选唯一当前播放器，屏外播放器无论是否预加载都不能胜出。
- `canonicalDouyinVideoUrl` 只从明确的 `/video/<id>` 或数字 `modal_id` 形成 `https://www.douyin.com/video/<id>`，不猜内容身份。
- 当前播放器有 HTTP 流时提交该完整流；只有 blob 时才建立页面解析候选。
- 直接流、视觉上下文和页面解析使用 `douyin:<videoId>`；标题读取当前 `feed-active-video` 的昵称与描述。
- 桌面在执行前再次规范旧 `modal_id`，并区分地址不支持与浏览器会话失效。

## 验收

- 自动测试以 223.237 秒主播放器和 49.041 秒屏外预加载器为固定回归，结果必须选择索引 0。
- 现场候选显示 3:43、当前画面和当前标题，不再出现 0:49 的“最佳可用”。
- 下载输出通过 FFprobe，时长与所选当前播放器一致；仅下载可打开目录，导入路线按选择进入 Eagle。
