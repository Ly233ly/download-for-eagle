## 修复目标

质量档位不是固定五项。每个媒体候选组必须只显示当前视频或其完整 HLS/DASH 主清单实际声明的质量，数量可以不同。弹窗显示“本视频 N 档”，避免把 Behance 示例的五档误解为产品限制。

## 已通过定向证据

- Vimeo 结构化目录：8 档输入完整保留并降序为 `2160p、1440p、1080p、720p、480p、360p、240p、144p`。
- 通用 HLS：从 8 条 `EXT-X-STREAM-INF RESOLUTION` 提取同一 8 档目录。
- 通用 DASH：从 `Representation height` 提取 `4320p、2160p、1080p`。
- popup 领域模型：显示当前组 8 档，最高/最低分别为 `2160p`/`144p`，不依赖固定集合。
- 桌面执行：8 个 ffprobe 节目中可准确选择 1440p、720p、480p，并在未指定时选择实际最高 2160p。

## 安全与回退

- 通用清单读取只在扩展内进行，不上传第三方；限制 2 MB、4 秒，并忽略扩展自身发起的捕获请求。
- 清单读取失败、仅为媒体清单或没有明确高度时，不生成猜测档位；保留“自动质量”并继续允许桌面 FFmpeg 处理。
- 签名 URL、Authorization 与 Cookie 不写入长期任务历史。
