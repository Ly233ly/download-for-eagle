# 扩展弹窗 UI 与交互逻辑重构规范

状态：1.2.3 单运行时与职责收口已实现；YouTube MSE/SABR 显示动态唯一高度，blob/分片页面显示带真实帧的稳定内容候选，重建分片只作技术后备；默认排除播放分片、最新项在底部、`completed_local` 可补导 Eagle，旧浏览器下载/解析/录制工具与上游图标已退出活动载荷  
范围：`chrome-extension/popup.html` 及其弹窗数据、选择、下载、合并、任务与工具交互  
原则：不复刻 cat-catch 的界面，也不保留其完整工具箱；扩展只做“看得见、选得对、提交真实”，本机软件负责“下载、合并、校验、保存、导入”。

## 1. 1.1.0 重构前问题诊断（历史）

这不是单纯的视觉问题。当时弹窗同时挂载了两套完整界面：

- `#eagleBridgeRoot` 维护一套候选、选择、配对、网站规则和本机任务状态。
- 原 cat-catch popup 继续维护当前页/其他页、筛选、选择、下载、合并、捕获工具和底部操作栏。

两套界面拥有不同的数据模型、选择状态和动作入口，导致以下结构性问题：

1. **同一资源有两套操作语义**：用户无法判断应使用新面板还是旧列表。
2. **资源没有按内容归组**：`currentGroup()` 把当前标签页的最多 200 条候选塞进一个伪组；广告、封面、字幕、清晰度和音轨可能混在一起。
3. **默认选择会跨内容配对**：`chooseDefaults()` 直接从全列表取最大视频和最大音频，可能把不同视频、广告或不同播放阶段的流合并。
4. **非法组合发现太晚**：清单与直链混选、DRM、缺少音轨等问题到点击下载后才报错。
5. **每次选择都整页重绘**：`root.innerHTML` 被完全替换，输入中的文件名、焦点、滚动位置和辅助技术上下文可能丢失。
6. **全局单任务状态不成立**：只保存一个 `planId` 和一条进度，无法正确表示批量任务、历史任务或同时进行的合并。
7. **主流程和高级工具混杂**：来源自动导入、忽略下一次、深度搜索、缓存捕捉、录制和下载动作争夺同一层级。
8. **动作没有明确结果**：本机下载、浏览器合并、停止、刷新并排出现，用户需要先理解实现方式再决定，而不是选择想要的结果。
9. **后端实际只合并第一路视频和第一路音频**：当前 UI 却允许任意多流进入同一方案，额外音轨或字幕会被下载但不一定进入最终文件。
10. **空状态不给路径**：提示用户开启某功能，但对应入口位于另一套 UI，发现与操作脱节。

## 2. 目标用户任务

弹窗按以下优先级设计：

1. 看见当前页面捕获到的“具体内容”，而不是一串 URL。
2. 一键下载推荐版本；B 站等分轨站点自动选择一条视频和一条音频并在本机无损合并。
3. 需要时展开选择清晰度、音轨、字幕和输出名；下载方式固定为本机软件。
4. 清楚看到每个任务正在下载、合并、导入还是失败，并能停止或恢复。
5. 未发现内容时，从同一个空状态启动增强捕获；不进入录制、移动 UA 或密钥工作流。
6. 网站来源记录和 Eagle 自动导入位于设置；浏览器下载器与外部下载器不进入产品流程。
7. “仅下载”完成后直接看见保存路径、打开所在文件夹或补导 Eagle；完成任务重开 popup 后仍显示 100% 和本机视频帧。

## 3. 新信息架构

弹窗只保留一棵 DOM 和一套状态源。

### 3.1 顶部栏

- 当前页面标题与域名；标题单行截断，悬停可看完整值。
- 捕获数量和本机助手连接状态。
- 刷新按钮与“更多”菜单。
- 不在顶部展示版本口号或实现名词。

### 3.2 主导航

- **媒体**：当前页面的内容级候选组，是默认页。
- **任务**：当前及最近下载/合并/导入任务，每项独立进度和操作。
- 主导航只保留 **媒体** 与 **任务**。唯一的增强捕获入口位于设置和媒体空状态，不创建第三条工具工作流。

### 3.3 媒体页

每个内容组包含：

- 可用时显示真实当前帧；页面帧只允许 URL 与播放器来源精确匹配，随后按 canvas 帧、跨域播放器矩形裁剪、poster、邻近内容图和公开页面元数据降级。仍无可信帧的普通视频预览当前所选下载直链自身，切换版本同步切换预览。站点 favicon、其他播放器或整页截图不得冒充视频封面。
- 内容标题、来源站点、时长、捕获时间。
- 推荐结果摘要，例如“1080P + 中文音轨 · 将本机无损合并”。
- 一个主按钮：`下载并导入 Eagle`。
- 次按钮：`仅下载`（不自动导入）。
- `选择版本` 展开面板：视频清晰度单选、音轨单选、字幕多选、输出文件名和方式。
- `技术信息` 折叠区：脱敏 URL、MIME、编码、请求来源、复制/二维码/清单查看等兼容能力。

同页多个内容组互不共享选择状态。批量模式通过显式“批量选择”开启，不把复选框永久暴露在每行左侧。

### 3.4 任务页

每个任务显示：

- 输出名称、缩略图和创建时间。
- 阶段：等待本机下载、本机下载/合并、校验、等待 Eagle、完成、失败、已停止。
- 真实本机百分比、已处理大小和阶段说明；未知总量使用媒体时长或阶段状态，不伪造字节总量。
- 活跃任务提供“停止”；失败任务提供“查看原因”和“重新创建方案”；完成任务提供“打开文件位置”或“在 Eagle 中查看”（接口可用时）。

任务列表来自 `plans`，不再由一个全局 `lastPlanId` 代表全部状态。

### 3.5 增强捕获

只保留一个按标签页启停的增强发现 Adapter：观察页面元素、Performance Resource、fetch 与 XHR 中的媒体 URL，并把结果送回同一媒体候选组管线。它不下载、不录制、不截取密钥、不改 User-Agent，也不连接第三方服务。启停通过刷新标签页恢复原生 fetch/XHR，实现边界清晰且可回滚。

### 3.6 设置/更多菜单

- 当前网站“记录来源并自动导入 Eagle”。
- 记录当前页、忽略下一次导入。
- 筛选和命名在媒体工作区就地完成；设置只保留来源规则、捕获总开关、独立窗口和增强捕获。
- 不显示浏览器/第三方下载方式、旧解析器入口、二维码或第二套媒体列表。

来源记录与下载选择是两个独立概念，网站规则关闭时下载仍可用。

## 4. 候选归组与展示模型

捕获层保留原始候选；视图层生成稳定的 `MediaGroupViewModel`，不得修改原始对象。

### 4.1 候选稳定标识

优先使用 `requestId`；缺失时使用以下字段的哈希：

`tabId + normalizedUrl + role + representation/label + capturedAtBucket`

URL 归一化仅用于去重/归组；执行下载仍使用原始 URL 和短时请求上下文。

### 4.2 硬边界

以下任一不同就不得自动合并为同一内容组：

- `tabId` 或主文档导航周期不同。
- 可识别内容 ID 不同（例如 B 站 `bvid/cid/aid/ep_id`）。
- 页面标题核心文本不同且捕获时间窗口不重叠。
- 明确的 manifest/master URL、播放器实例或上游任务 ID 不同。
- 资源被识别为广告/追踪/图片且不是内容封面。

### 4.3 软归组

没有站点专用内容 ID 时，使用以下证据评分：

- 同一页面 URL/initiator/referer。
- 标题核心文本相同。
- 捕获时间接近。
- 时长相近。
- URL 路径/查询中的公共表示键相同。
- 视频、音频、字幕角色互补。

低置信度候选进入“未归组资源”，绝不自动与推荐视频合并。用户手动归组只影响本次会话。

### 4.4 组内分类

- `videoVariants`：按高度、码率、帧率、编码和容器排序，单选。
- `audioTracks`：按语言、标签、码率和编码排序，单选。
- `subtitleTracks`：按语言和格式排序，多选；在后端未支持封装前明确标记为“单独下载”。
- `manifests`：HLS/DASH 清单与直链轨道互斥；选择清单后走本机清单路由或专用解析器。
- `attachments`：封面、图片、JSON 和其他附件，不参与自动音视频合并。

### 4.5 推荐规则

推荐必须在一个内容组内计算：

1. 排除 DRM、广告、空 URL、已知不可下载和低置信度跨组资源。
2. 有完整直链媒体时，推荐一个最高可用且兼容的完整媒体。
3. 有分轨时，推荐一条最高质量视频和一条首选语言/最高质量音频。
4. 只有 HLS/DASH 清单时，推荐本机清单处理。
5. 未知大小不视为 0 质量；优先使用分辨率、码率、标签和编码。
6. 推荐结果必须在按钮上方可读，不能静默改变。

## 5. 本机统一下载决策

用户默认选择结果，系统解释使用的方式：

| 选择 | 本机动作 | 结果 |
| --- | --- | --- |
| 单个普通直链 | 本机 FFmpeg 下载/封装 → ffprobe | 导入 Eagle 或 `completed_local` |
| 一条视频 + 一条音频 | 本机 FFmpeg 同时拉取 → streamcopy → ffprobe | 一个最终媒体 |
| HLS/DASH 清单 | 本机 FFmpeg 解析、下载、streamcopy → ffprobe | 一个最终媒体 |
| 多个独立资源 | 每个内容组创建独立 `desktop` 任务 | 批量导入或批量仅下载 |
| 字幕 | 本机软件单独下载为 sidecar | 不伪装成已封装进视频 |
| `blob:` | 创建任务前阻断 | 指导选择真实直链/清单，不回退浏览器 |
| DRM | 阻断并说明 | 无绕过或解密入口 |

前端在动作可用性计算阶段就拒绝清单/直链混选、跨组音视频、两个视频轨或两个音轨进入同一合并任务。

## 6. 单一状态模型

```text
PopupState
├─ connection: unknown | connecting | paired | needs_pairing | offline
├─ activeTab: { id, navigationKey, url, title, favIconUrl }
├─ capture: loading | ready | empty | refreshing | error
├─ candidatesById: Map<CandidateId, Candidate>
├─ groupsById: Map<GroupId, MediaGroupViewModel>
├─ groupOrder: GroupId[]
├─ selectionsByGroup: Map<GroupId, GroupSelection>
├─ draftsByGroup: Map<GroupId, { outputName, importToEagle }>
├─ tasksById: Map<PlanId, TaskViewModel>
├─ framePreviews: Map<PlayerGroupKey, DataUrl>
├─ advancedCapture: { open, toolsById, possibleKeys }
└─ siteRule: loading | enabled | disabled | error
```

状态更新采用小范围 patch：新增候选只更新所属组，勾选只更新当前组和动作摘要，任务轮询只更新对应任务。不得再用整页 `innerHTML` 重绘。

需要保留的瞬时状态：当前标签、滚动位置、展开项、焦点、输出名草稿、每组选择、批量选择和错误详情。  
需要持久化的状态：捕获/筛选设置、网站规则、工具开关、任务引用。Cookie、Authorization 和完整敏感请求头不得进入持久状态。

## 7. 操作状态机

### 7.1 启动

`boot → connection_check → tab_load → candidates_load + site_rule_load + tasks_load → ready`

任何分支失败只影响对应区域：本机助手离线时仍可浏览候选和复制技术信息，但“下载并导入”和“仅下载”都禁用；站点规则失败不应清空媒体；任务读取失败不应清空已显示候选。

### 7.2 内容组

`detected → grouping → recommended → customized → validating → actionable`

异常分支：

- `blocked_drm`：禁用下载并显示原因。
- `ambiguous`：不自动合并，要求用户选择/归组。
- `unsupported`：展示原因；只有传输分片时要求继续播放/深搜等待清单，不提供外部下载回退。
- `stale`：刷新候选并保留仍存在的选择。

### 7.3 任务

`creating → queued → downloading → validating → ready_to_import → imported`

仅下载终态：`completed_local`。终止分支：`retry`、`failed_permanent`、`canceled`。  
停止按钮只对当前任务生效；完成、失败或取消后必须停止轮询。多个任务使用一个调度器批量刷新，不能为每项无限创建独立定时器。

## 8. 错误与恢复

| 情况 | 用户看到的结果 | 恢复动作 |
| --- | --- | --- |
| 本机助手离线 | 顶部离线；所有下载动作禁用，候选仍可查看 | 重新连接/配对 |
| 当前页没有媒体 | 解释需要先播放；提供增强捕获 | 启动后刷新页面并原地显示状态 |
| 清单与直链混选 | 选择面板内即时提示，不允许提交 | 切换为清单或直链方案 |
| 跨内容音视频 | 不自动归组；显示“来源不确定” | 手动选择同组轨道 |
| 本机下载失败 | 对应任务显示 FFmpeg/网络错误，不覆盖其他任务状态 | 同次运行重试；上下文失效时回来源页重建 |
| FFmpeg streamcopy 失败 | 显示容器/编码原因 | 更换容器；用户确认后转码 |
| Eagle 不可用 | 最终文件保留，任务停在待导入 | Eagle 恢复后重试导入 |
| DRM | 清晰阻断 | 无下载、解密或绕过入口 |

所有错误必须关联到具体内容组、媒体流或任务，禁止只在弹窗底部显示一条全局红字。

## 9. 无障碍与视觉约束

- 键盘可完成主导航、展开、版本选择、下载和停止。
- 所有图标按钮有可见提示和 `aria-label`；状态不只依赖颜色。
- 焦点在局部更新后保持；错误通过 `aria-live` 通知但不抢焦点。
- 文本和控件满足 WCAG AA 对比度；点击目标至少 32×32 CSS px。
- 弹窗最窄宽度下标题、元数据和主按钮不溢出；任务进度不挤压停止按钮。
- 不使用 emoji、上游猫图或文本符号充当正式图标；只使用下载中转站自有且风格一致的 SVG/PNG 资产。
- 视觉方向必须基于当前弹窗实机截图生成三套独立方案后选择，不能直接复制 cat-catch。

## 10. 验收标准

### A96 单界面与单状态源

- `popup.html` 只有一个可见应用根节点和一套主导航。
- 新旧选择、下载、筛选和工具入口不重复出现。
- 选择、输出名、滚动和焦点不会因新候选或任务刷新而丢失。

### A97 内容级归组

- 同一 B 站视频的视频/音频轨进入同一组；广告或另一内容不会被自动合并。
- 低置信度资源进入“未归组”，不会参与推荐。
- 每组推荐规则可用纯函数测试覆盖，未知大小也能正确排序。

### A98 可见选择与路由

- 默认可见标题、封面/无预览原因、清晰度、音轨、大小、输出名和预计动作。
- 清单/直链混选、跨组轨道、DRM 在提交前阻断。
- B 站推荐动作明确写出“下载视频 + 音频并在本机无损合并”。

### A99 多任务与恢复

- 同时创建两个任务时各自显示阶段、进度、错误和停止按钮。
- 切换主标签、关闭再打开 popup 后能从 `plans` 恢复任务列表。
- popup 首次打开即等待 session 媒体快照恢复完成，后续缓存变化自动刷新；工具栏角标与媒体列表使用同一内容归组口径。
- 任务完成/失败/取消后停止轮询，不再显示可用的停止按钮。

### A100 功能对等重排

- 这是 1.1.x 历史验收，已由 A145–A151 取代。1.2.0 只保留捕获、筛选、内容帧、增强发现、设置和独立窗口；缓存代理、录制、移动 UA、密钥、媒体控制、二维码、手工解析器、旧自动下载和外部目标均明确退役。

### A101 视觉与交互 QA

- 使用用户提供的当前 popup 截图完成问题标注。
- 先产出三套相互独立的视觉方向并由用户选定，再进入正式实现。
- 在 Chrome 实际扩展 popup 中覆盖：加载、空、单直链、B 站分轨、HLS/DASH、DRM、离线、下载中、合并中、失败和完成状态。
- 参考图与实现截图使用相同视口并排比较，修正裁切、间距、字体、边框、控件状态和滚动问题。

### A102 回归与交付

- 原有 68 项测试继续通过，并新增归组、推荐、动作可用性、任务归一化和 DOM 交互测试。
- Chrome/Firefox 清单和全部 JavaScript 语法检查通过。
- 重新构建安装包、覆盖安装、验证健康接口与扩展协议，再更新安装/回滚/发布证据和哈希。

## 12. 方案 2 实现记录

- 采用 660×574 的浅色双栏弹窗：左侧是内容组，右侧是版本/音轨/输出与主动作检查器。
- 左侧不把视频轨与音轨伪装成两个“内容”；B 站同一 `bvid/cid` 的轨道在右侧选择并由本机 FFmpeg 合并。
- `popup.html` 只有一个应用根和一个新版运行时；隐藏上游 DOM、旧 popup 与媒体控制脚本已删除。
