# 关键决策与风险

## 当前决策

1. **来源可选**：有可靠网页就写入，没有也直接导入，不猜测。
2. **网站规则采用明确事件**：扩展识别到关闭网站的下载动作时发送跳过事件；无浏览器证据时不能仅凭 IDM 文件判断网站。
3. **保留原文件**：成功后不移动、不删除、不修改下载视频。
4. **导入“全部”**：不传文件夹编号，不自动加标签。
5. **离线可恢复**：任务先入库；Eagle 关闭、助手退出或系统重启不会丢任务。
6. **重复按完整内容判断**：使用 SHA-256，文件名和路径不作为重复依据。
7. **接收器轻量化**：IDM 进程只负责入队和唤醒，耗时工作全部在助手后台。
8. **只使用官方 Eagle API**：导入和补来源都通过本地接口，不碰资源库内部文件。
9. **本地优先**：没有云账号、遥测、外部同步或网页正文采集。
10. **个人分发版**：扩展以“加载已解压”方式安装，不伪造企业策略或绕过 Chrome 的用户确认。
11. **托盘进程隔离**：系统托盘由 .NET Windows Forms 宿主管理；Python 不得再用 `ctypes` 替换窗口过程，宿主与助手只通过命名事件通信。
12. **自带运行环境**：分享包使用 PyInstaller `onedir`，避免要求非技术用户安装 Python，也避免 onefile 每次 IDM 通知都解压。
13. **下载触发启动**：不默认写入 Windows 开机启动；IDM 下载完成时发现助手未运行再静默启动，降低常驻占用。
14. **一次性自动配对**：安装器生成机器级随机秘密，保存哈希且七天过期；Chrome 扩展换取正式令牌后立即销毁。
15. **保护已有安全软件**：IDM 病毒检查入口非空且不是本助手时拒绝覆盖，便利性不能高于用户原安全配置。
16. **自有进程标识**：正常运行统一使用带版本资源和自有图标的“下载中转站”主程序与冻结后台，不让用户在任务管理器中看到属于本助手的通用 Python 进程。

## 1.0.0 已实施决策

17. **一次性功能对等发布**：可以按任务独立开发和验证，但只有 cat-catch 对照矩阵、A46–A95 和旧功能回归全部通过后才发布 1.0.0。
18. **候选组而非 URL 列表**：一个用户可识别媒体对应一个媒体候选组；DASH 视频、音频和字幕是组内流，不是互不相关的下载项。
19. **下载前默认可见**：标题、封面或预览原因、类型、清晰度、音轨、大小、路由和预计输出默认展示；长 URL 退居可展开技术信息。
20. **本机 FFmpeg 默认**：独立音视频默认在本机 streamcopy 并用 ffprobe 验证；可避免重编码时不得为了统一流程强制转码。
21. **浏览器 FFmpeg 是显式兼容项（已由决策 37 取代）**：1.0.0 曾保留该高级入口；1.1.0 本机统一下载后不再出现在可见界面。
22. **浏览器负责会话型下载（已由决策 37 取代）**：早期组件路径回传方案已删除，扩展不再负责媒体数据下载。
23. **DRM 只检测和阻断**：不迁移、不实现、不测试 Widevine、SAMPLE-AES 等 DRM 绕过；预览失败与 DRM 阻断必须区分。
24. **临时文件必须有归属**：只有本机软件创建、位于 `下载中转站/临时/<planId>` 的明确中间文件可以清理；现有“保留原文件”原则继续适用于用户下载。
25. **外部下载目标退出可见主流程**：第三方在线 FFmpeg、Aria2、m3u8dl、MQTT、send2local、自定义命令、自动下载和完整浏览器下载器不进入新版可见界面。
26. **GPL-3.0 组合发行**：1.0.0 已采用 GPL-3.0 组合发行，保留项目 MIT 与所有上游通知，并把固定 cat-catch 源码快照、构建脚本和获得源码方式放入发行包。
27. **重做而非复刻 popup**：cat-catch 的功能实现继续保留，但不复制其 popup 信息架构；1.1.1 的弹窗只保留一套 DOM、状态源和“媒体、任务”主导航，高级捕获集中在设置中。
28. **先归组再推荐**：推荐视频和音频必须属于同一内容组；低置信度资源不自动合并，清单/直链混选和 DRM 在提交前阻断。
29. **用户选择结果而非实现**：主动作描述最终结果（下载、合并、导入），浏览器/本机/外部工具作为系统解释或高级选择，不能并排要求用户先理解技术路线。
30. **任务状态按方案隔离**：popup 从本机 `plans` 恢复多任务状态，每个方案独立进度、错误和停止；不再用单个 `lastPlanId` 代表全部操作。
31. **方案 2 作为 1.1.0 视觉基线**：采用浅色双栏工作台，左侧显示内容组、右侧选择版本与输出；视觉稿中的“视频/音频两条内容”纠正为同一内容组内的轨道选择。
32. **隐藏兼容运行时而非第二界面（已由决策 44 取代）**：1.1.0 曾保留隐藏 DOM 适配层；1.1.1 已改为新版 UI 直接调用后台/内容脚本能力并删除隐藏 popup 运行时。
33. **逐下载请求头规则（已由决策 37 取代）**：Chrome DNR 组件规则及其恢复测试、状态和接口已删除。
34. **受保护分轨优先本机（扩展为决策 37）**：B 站问题证明本机 FFmpeg 路线可靠；1.1.0 将该路线从“受保护分轨优先”扩大为所有媒体唯一主路线。
35. **媒体计划与 Eagle job 共用终态**：`jobs.imported` 必须在同一数据库事务中把关联 `download_plans` 更新为 `imported/100`；列表读取同时修复旧版本遗留的不一致记录，popup 重开只相信本机持久状态，不用内存中的旧进度猜测。
36. **内容图与站点身份分离**：favicon 只表示网站，不得进入媒体 `thumbnailUrl`。1.1.1 优先抓取真实视频帧；跨域时截取播放器可见矩形，poster、邻近内容图和公开元数据只作回退。瞬时帧只在扩展内存中存在。popup 首次读取必须等待 session 快照恢复，工具栏和展开列表必须复用同一内容归组口径。
37. **所有主下载统一由本机软件执行**：扩展只发现、归组、展示和提交任务；普通直链、分离音视频、HLS/DASH、“下载并导入”和“仅下载”均固定 `route=desktop`。禁止静默回退 Chrome 下载器、浏览器 FFmpeg 或第三方目标。
38. **下载与导入是两个明确结果**：`import_to_eagle=1` 只有在媒体下载、合并和 ffprobe 校验成功后创建 job；`import_to_eagle=0` 以 `completed_local/100` 结束，不触发 Eagle。
39. **敏感媒体上下文只在本机进程内存**：完整签名 URL、Cookie、Authorization、Referer、Origin 和 User-Agent 不写 SQLite 或扩展持久存储。同次运行可以重试；进程重启后明确要求回来源页重建，不用脱敏 URL 猜测。
40. **schema 5 删除组件交接模型**：`component_files`、组件完成/失败 API、Chrome downloadId、浏览器进度聚合和 DNR 规则不再属于领域模型；方案直接记录本机阶段、字节数、预览、最终路径与关联 job。
41. **桌面与扩展共用媒体计划事实源**：扩展任务页和桌面“媒体下载”页都读取 SQLite `download_plans`；下载完成只到 90%，关联 Eagle job 进入 `imported` 后才显示 100%，重开任何界面不靠旧内存猜进度。
42. **不可本机访问的 URL 明确阻断**：`blob:` 等浏览器私有地址在创建方案前失败并指导选择真实直链或 HLS/DASH 清单，不以“兼容”为名恢复浏览器下载分支。
43. **所有可达媒体下载表面使用同一协议**：popup、预览页、独立直链页和 HLS/DASH 解析页最终都发送认证 `createPlan`；旧自动下载快捷键/右键项/后台状态和捕获后自动外发删除。浏览器只可保存录制等本地生成 Blob，不得用它下载远程候选 URL。
44. **popup 只有一个运行时**：`popup.html` 只加载 `eagle-bridge-ui-logic.js` 与 `eagle-bridge-ui.js`；旧 `popup.js`、`media-control.js`、隐藏 DOM、jQuery/MQTT/HLS popup 依赖和其第二状态源删除。高级能力由新版 UI 直接发送明确消息。
45. **同一播放器优先于轮换请求 URL**：页面为每个播放器生成短时稳定视觉键；URL 归一化只用于同一播放器内的展示去重，真实原始 URL 仍用于本机任务。签名查询变化不得制造新内容卡片，不同播放器或明确内容 ID 仍禁止混组。
46. **传输分片不是可下载内容**：非显式分轨的 `.m4s`/`.ts` 片段在同组有清单/完整媒体时隐藏；仅捕获到分片时只保留一个代表并阻止创建任务，提示继续播放、深搜或等待清单，不能让用户盲选片段。
47. **真实帧是瞬时展示数据**：同源/允许 canvas 时从 `video` 生成 JPEG；跨域受限时由后台对当前活动标签页的播放器矩形截图裁剪。帧有尺寸、格式和长度门禁，只在 service worker/popup 内存缓存，绝不进入 session 媒体快照、本机任务、SQLite 或 Eagle 来源字段。
48. **清单优先于自适应分片**：同一稳定播放器组出现完整 HLS/DASH 清单后，非显式分轨的 MP4/音频请求视为清单内部传输分片，不再作为独立版本供用户下载。
49. **播放分辨率与文件质量分离**：`video.videoWidth/videoHeight` 表示播放器当前输出，只能显示为“当前播放”；只有结构化目录、清单属性或明确流元数据才能形成可下载质量档位，禁止把当前播放 1080p 复制给每个网络请求。
50. **清晰度选择由本机节目映射执行**：扩展传递规范化的 `preferredQuality`，本机 ffprobe 选择最接近目标高度的节目视频及关联音频，FFmpeg 只输出这一档；未选择时取最高可识别档。
51. **质量档位数量由当前视频决定**：不维护固定五档、常见档位或站点默认表；只保留当前结构化目录或主清单明确声明的高度，去重降序后显示动态数量。
52. **通用清单发现必须有界且可失败**：扩展读取 HLS/DASH 质量目录限制为 2 MB 和 4 秒，复用浏览器会话且不上传第三方；失败时不猜测、不阻断本机 FFmpeg 路径。
53. **任务完成态优先于旧进度字段**：`completed_local` 与 `imported` 的领域含义已经确定输出完成，popup 必须显示 100%；轮询失败只能标记同步中断，不能继续把创建时的 0% 当作实时事实。
54. **失效配对必须可恢复**：认证接口返回 401 时扩展清除失效令牌并尝试安装器 bootstrap；每次成功覆盖升级在健康门与系统集成完成后轮换新的单次凭据，失败回滚前不轮换。无法自动恢复时显示需要配对，不把认证错误永久伪装成助手离线。
55. **任务预览按能力回退而非站点白名单**：下载前使用通用 video 当前帧、播放器可见矩形、poster/媒体元数据；下载后使用本机 FFmpeg 抽帧。站点适配可以补充结构化质量目录，但不得决定任务是否显示视频图像。
56. **本机可达动作只接受任务身份**：打开输出和读取预览只允许客户端提供 `planId`；服务端从数据库解析文件并校验属于程序的完成/预览根，禁止客户端路径参数。
57. **认证失败只能条件清除令牌**：401 只能清除发起该请求时使用且仍为当前值的令牌；若配对或恢复已写入更新令牌，旧请求必须保留新状态并重试。UI 只有在受认证健康检查成功后才能宣布配对完成。
58. **扩展共享状态必须串行读改写**：`downloadTransferStation` 的令牌、事件队列和最近任务字段属于一个一致性单元；所有更新通过同一队列读取最新快照后提交，事件更新与令牌清除使用函数式补丁，禁止并发旧快照整对象回写。
59. **受认证读取统一使用 POST**：Chrome 对扩展跨源 GET 的 Origin/Authorization 头可能与配对 POST 不一致；健康、任务列表、详情和预览改用 JSON POST，让随机令牌进入请求体回退，同时继续验证浏览器扩展 Origin。旧 GET 只保留兼容，不再由当前 popup 调用。
60. **视觉身份只能来自真实 URL 匹配**：只有媒体请求与播放器 `currentSrc/src/source` 在去掉签名查询后仍为同主机、同路径，才允许继承播放器帧、时长、尺寸和稳定组键。当前播放或可见播放器不能作为未匹配请求的身份回退；宁可显示未确认候选，也不能把后台预加载视频描述成当前视频。
61. **最终时长是内容身份安全门**：非清单直链的声明时长达到 5 秒时，FFprobe 输出与声明相差超过 `max(3 秒, 8%)` 必须阻止交付和 Eagle 导入。该门禁是捕获身份关联的第二层保护，不用于替代 URL 匹配，也不对直播/清单猜测固定时长。
62. **缺少可信帧时预览候选自身而非其他播放器**：普通视频候选没有精确页面帧或结构化封面时，popup 可直接解码当前所选 `http/https` 媒体 URL 作为只读预览；预览源必须与当前下载选择共用同一候选对象。加载失败显示占位，不得回退到其他播放器、整页截图或站点专用图片。
63. **CDN 备用地址只按强媒体别名归并**：同一标签页和来源网站、同媒体类型下，响应 Digest/Content-MD5/强 ETag 相同，或长媒体路径相同且总字节数完全一致，才可把不同 URL 视为同内容别名。文件大小、标题、捕获先后或当前可见播放器不能单独证明同一内容；别名指向多个显式播放器组时必须保持隔离。
64. **URL 自身固定的字节区间属于传输分片**：查询、路径明文或 base64url 路径段明确编码 `range/bytes=start-end`，且候选大小与区间长度一致时，即使后缀和 MIME 为 MP4 也不能作为完整媒体提交。扩展负责隐藏/阻断，桌面负责独立复验；不能把某个站点域名、单次 HTTP 206 或 FFmpeg 失败作为判据。
65. **1.2.0 以职责边界取代上游工具对等运行时**：扩展模块只实现网络/页面媒体发现、媒体候选组、真实预览、筛选、配对、来源规则、任务提交和状态控制；桌面模块独占远程下载、HLS/DASH 解析执行、音视频合并、FFprobe 校验、文件持久化和 Eagle 导入。旧下载页、解析页、录制、移动 UA、密钥、浏览器 FFmpeg 和第三方路由全部从活动载荷删除，决策 27、43、44 中对这些兼容入口的保留描述由本决策取代。
66. **播放分片默认退出信息主线**：`segmentOnly` 组默认从媒体列表排除并显示隐藏数量；用户只能通过“显示不可下载播放分片”筛选项查看 64px 紧凑诊断行，详情不显示大预览且永远不能创建下载方案。
67. **候选按捕获时间正序、入口跟随最新项**：最早内容在上、最新内容在下；首次打开或用户原本停留在最新项时，自动选中并滚动到底部。用户主动查看旧内容时，新捕获不得抢走选择。
68. **本地完成文件允许补导而不重下载**：`completed_local` 且最终路径属于程序 `已完成` 根时，可以幂等创建 Eagle job，状态转为 `ready_to_import/90`。API 和桌面/UI 只接收 `planId`，服务端重新解析并校验路径归属。
69. **统一 GPL-3.0 发行但保留法定来源记录**：1.2.0 运行时使用下载中转站名称与自有图标，删除上游 UI/图标与未使用浏览器库；因仍有修改后的 GPL 来源文件，固定 cat-catch 源码、版权与修改记录继续随源码/发行物提供，不能以“改动较多”为由移除。
70. **MSE 站点优先读取结构化播放器目录**：当 DOM `currentSrc` 和网络请求只暴露占位地址/播放探测时，允许站点适配器在页面主世界读取播放器已经生成的格式目录，并以明确内容 ID 和流 ID 形成通用媒体候选组。YouTube 以 videoId/itag 实现；不能按 CDN 路径、文件大小或当前画面猜身份。
71. **结构化目录只接受直接媒体 URL**：只有播放器目录明确给出的 `http/https` 地址进入候选。只有加密签名字串、DRM 描述或需要逆向解签的格式不得伪造成可下载项；本项目不实现签名或 DRM 绕过。目录发现仍属于扩展职责，下载、合并和校验仍独占于桌面。
72. **小型无身份音视频默认视为传输探测**：URL 显式携带固定 Range 时，不因 Content-Range 暴露完整文件总长而改变分片性质；无 groupKey、无时长且小于 128 KB 的音视频候选也进入诊断分区。明确结构化分组的小音轨或初始化信息不受该启发式误伤。
73. **SABR 元数据目录不是直链但可以成为桌面解析方案**：决策 71 继续禁止把无 URL 格式伪造成普通可下载项；当播放器明确同时提供 videoId、多档自适应视频高度和 `serverAbrStreamingUrl` 时，扩展可建立 `resolver=youtube` 目录候选。用户只选择唯一高度，桌面解析器必须证明该高度可用后才进入统一下载状态机。
74. **YouTube 解析依赖固定本机工具和瞬时认证交接**：使用固定哈希的 yt-dlp/Deno，不在浏览器实现 SABR、签名或媒体下载。Cookie 不进入参数、数据库、日志或诊断；为兼容 yt-dlp 只能在程序拥有的 `临时/<planId>` 创建最小 Netscape cookie 文件，并在解析进程结束的 `finally` 中删除。该受控例外不允许扩展到用户 Cookie 导出或长期 cookie jar。
75. **字节偏移参数先按传输语义处理**：成对 `bytestart/byteend` 与 `range/bytes/start/end` 都是固定区间声明，不能因 `.mp4` 后缀或响应 MIME 直接作为完整文件。能够重建时只删除偏移参数并保留签名；不能证明时保持分片阻断。桌面仍独立复验，规则不依赖 Instagram 或任何站点域名。
76. **blob 页面以稳定内容地址建立解析候选**：顶层页面中的 blob 视频只有在当前 URL 或最近内容容器存在同源、可识别的内容永久链接时，才可形成 `resolver=page`；主页、搜索页或没有内容 ID 的 feed 不允许猜测。链接规范化只保留决定内容身份的安全查询键。
77. **通用解析器取代逐站下载器**：Instagram、TikTok/抖音、X、Reddit、Facebook、Pinterest 等页面共用固定 yt-dlp/Deno 解析入口和统一 FFmpeg 状态机；站点适配只负责提供可信内容身份或结构化元数据，不复制下载、合并、进度和 Eagle 逻辑。解析器不支持、登录失效或 DRM 时显式失败。
78. **稳定页面候选优先于重建传输后备**：同一标签页与来源域已经存在 `resolver=page` 时，字节分片重建得到的候选默认进入技术分片分区，避免用户面对同一内容的页面候选和多个 CDN 轨道；用户显式显示技术分片时仍可审计后备地址。
79. **抖音适配只确定当前内容身份**：精选页允许一个独立适配器用视口交集、播放状态、播放进度和面积选择唯一主播放器，并从明确 `/video/<id>` 或数字 `modal_id` 建立 `douyin:<videoId>`。它不能复制下载、解析、进度、合并或 Eagle 状态机；屏外预加载项不得继承当前内容身份。
80. **抖音当前 HTTP 流优先于页面解析**：当前主播放器给出完整 HTTP `currentSrc/src/source` 时直接交统一本机下载；只有 blob 且存在明确内容永久链接时使用 `resolver=page`。无内容 ID 的 `/jingxuan` 既不能猜成详情页，也不能作为可重试候选；地址不支持和浏览器会话失效必须使用不同错误语义。
81. **抖音标题属于 feed-item 而非标签页**：每个已加载播放器只能从自己的明确视频 ID、作者节点和文案节点形成内容候选；当前项与预载项允许同时显示，但不得跨容器继承元数据。blob/MSE 底层请求若不能精确绑定播放器和视频 ID，只能作为隐藏技术分片，不能用标签页标题补成可下载内容。
82. **内容绑定候选优先于未关联播放请求**：信息流中每个 blob/MSE 播放器若能从最近内容容器证明同源永久链接，就形成自己的页面解析候选，并只读取该容器的标题、正文、封面和时长。同一标签页/来源域存在这些内容候选时，无稳定组键/内容身份的孤立 HLS、DASH、音视频请求退出默认主线、只作不可提交技术信息；没有内容候选的唯一完整清单仍保持可用。规则不得按复现网站域名分支。
83. **页面发现必须可恢复且不可被 DOM 变化饿死**：信息流 MutationObserver 只能使用有界合并调度，不能用可无限后移的尾随防抖。popup 读取候选前必须探测当前 HTTP(S) 标签内容脚本；接收端缺失时只重新注入候选逻辑和内容发现脚本并立即扫描。该恢复不允许注入下载器、站点代码、受限协议页面或其他标签。
84. **页面分轨按容器能力逐级选择**：通用解析优先 MP4 视频+M4A 音频，同时接受 MP4 视频+MP4 音频；无首选组合时才回退到合并 MP4、MP4 视频+任意音频和通用最佳格式。轨道扩展名不能代替媒体角色判断，不为复现站点增加下载分支。
85. **结构化视频元数据可以证明内容入口**：网页明确声明视频类型/Player 卡片并给出同源、非首页的规范地址时，可以在没有可读播放器 DOM 的情况下形成 `resolver=page` 候选。该证明不能扩展到普通网页、主页或跨源规范地址，也不能把播放器底层未关联请求提升为内容。
86. **标签页和域名不是内容身份**：一个 `resolver=page` 只证明其自身内容可解析，不能据此隐藏同标签页、同来源域名的其他完整 MP4、音频、HLS 或 DASH。展示层只默认排除结构已确认的传输分片；blob/MSE 播放器按 DOM 距离选择最近的稳定作品链接，没有更近链接时保留当前作品页。该规则不得按 Pinterest 或其他复现站点域名分支。
87. **桌面下载线路显式跟随系统代理**：Chrome 使用 Windows 系统代理不代表 FFmpeg 或 yt-dlp 会自动继承。桌面必须在每个任务开始时读取当前用户代理，并显式传给所有远程输入；不得把全进程环境或本机 Eagle/API 一并代理。
88. **自动模式最多一次线路回退**：检测到系统代理时先使用代理，网络失败只允许改用直连一次；没有代理时只直连。手动模式代表用户明确选择，失败时不得静默绕过。该边界防止无限重试和隐私意外。
89. **统一代理只接受 HTTP/Mixed 端口**：FFmpeg 的统一输入代理使用 HTTP CONNECT；手动设置因此只接受无认证的 HTTP/Mixed 主机端口。只提供 SOCKS、浏览器扩展代理或 PAC 且无法自动检测时，由用户在代理软件中启用 HTTP/Mixed 端口，不引入浏览器高权限代理读取。
90. **代理诊断最小化**：代理模式、检测来源和主机端口可以进入诊断，账号、密码和完整代理 URL 不得保存或导出。403/超时只提供可操作线路提示，不将代理问题误报为缺少 FFmpeg。

## 来源与网站规则的边界

IDM 只把完成文件路径传给接收器，不提供网页地址或 Chrome 标签页。因此“无来源也导入”与“按网站严格阻止”无法在所有场景同时保证。当前折中是：

- 扩展捕获到下载点击时，开启网站记录来源，关闭网站发送跳过。
- 视频播放产生的普通媒体请求不会在关闭网站上自动发送跳过，避免误伤。
- IDM 浮动按钮如果没有被网页脚本识别，又没有媒体文件名线索，任务按无来源直接导入。
- 用户始终可以在下载前点击“本次下载不导入”。

## 隐私与安全

- 保存任务状态、文件路径、来源网页、域名、时间、指纹、Eagle 项目编号和有限错误信息。
- 不长期保存 Cookie、密码、Authorization、请求头、完整签名媒体 URL、网页正文或完整浏览历史；所需最小请求上下文只在经过配对认证的当前本机任务内存中存在，终态即清理。
- 清理常见广告跟踪参数；诊断导出只包含文件名和来源域名，不包含完整路径与网址。
- 本机接口仅监听回环地址，并校验扩展 Origin 与随机令牌。
- 安装器配对文件只保存随机秘密的 SHA-256，成功配对后删除；发行包本身不含任何用户令牌或数据。

## 已知风险

- 大视频完整哈希会产生顺序磁盘读取，但在后台单线程和固定缓冲区内执行，不阻塞 IDM 与界面。
- Eagle 能否预览某种视频取决于容器与编码；扩展名受支持不等于所有编码都可预览。
- IDM“病毒检查”入口可能被安全软件占用；有原配置时不能覆盖。
- 未上架 Chrome 应用商店，因此接收者仍需亲自确认“加载已解压的扩展程序”。
