# 1.2.11 系统代理跟随验证报告

日期：2026-07-20

## 结论

1.2.11 已完成浏览器与桌面下载出口统一。默认“自动”模式会在每个新任务开始时读取当前用户的 Windows 系统代理，并把同一线路用于 FFmpeg/ffprobe、yt-dlp 和直接字幕请求；如果代理软件只接管 Chrome、没有写入 Windows 系统代理，用户可在软件的“网络”设置中填写 HTTP/Mixed 地址，例如 `127.0.0.1:7890`。

本机 Eagle、配对、本地 API 和控制信号始终直连。自动模式检测到代理时先走代理，只有网络类失败才切换直连一次；手动模式和直连模式均不增加隐藏线路，因此不会无限重试。

## 关键边界

- 自动、直连和手动三种模式均会持久保存，但软件不会修改 Windows 或代理软件的设置。
- 手动地址只接受无账号密码的 HTTP/Mixed 端口；诊断只导出模式、检测来源和脱敏主机端口。
- 直链、HLS/DASH、页面解析、YouTube 解析和字幕请求共用任务级代理选择；直连模式会显式禁用 Python 的隐式系统代理。
- 403、超时和连接失败会提示检查浏览器与桌面代理是否一致，并指向“网络”设置。

## 接收者使用

一般情况下保持“网络：自动”即可。如果 Chrome 能打开 Behance 等网站，但下载仍报 403，说明代理可能只接管浏览器：打开软件主界面的“网络”设置，选择“手动代理”，填写代理软件显示的 HTTP/Mixed 地址与端口并保存，再重试失败任务。无需开启 TUN，也无需单独安装 FFmpeg 或 Python。
